## Development

When starting the dev server, use background mode:

```
astro dev --background
```

Manage the background server with `astro dev stop`, `astro dev status`, and `astro dev logs`.

## Project Rules

- Do not invent technical specifications, performance figures, measurements, model names, certifications, or commercial claims.
- Use official or explicitly approved sources for product information.
- Do not copy manufacturer text literally; summarize and rewrite with attribution when needed.
- Do not claim official, exclusive, or authorized representation unless the business confirms it.
- Keep business data and presentation separate. Edit general company data in `src/site.config.ts`.
- Do not add dependencies without a clear justification.
- Keep all visible user-facing text in Spanish.
- Use English for internal file names, variable names, component names, and technical comments.
- Prioritize accessibility, performance, responsive design, and SEO.
- Run `npm run build` after relevant changes.
- Clearly mark all provisional content.
- Do not download external images, logos, videos, or documents unless the source and license or authorization are recorded.
